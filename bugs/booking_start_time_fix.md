# Bug: Single Booking Detail Overwrites `start_time`

## Bug ID
LEFTOVER-001

## Summary
The endpoint `GET /bookings/{booking_id}` returned the wrong `start_time` in its response payload because it overrode the serialized start time with the creation time `booking.created_at`.

## Severity
High

## Rule Violated
Single booking detail response must return the correct `start_time` value matching the reservation time.

## Affected Files
* [app/routers/bookings.py](file:///d:/ICT_Fest_Hackathon_Preliminary/app/routers/bookings.py)

## Modified Line Numbers
* `bookings.py` : Line 179

## Original Code
```python
    response = serialize_booking(booking)
    response["start_time"] = iso_utc(booking.created_at)
```

## Fixed Code
```python
    response = serialize_booking(booking)
```

## Root Cause
An erroneous line of code manually reassigned the response's `start_time` key to match the value of `booking.created_at`.

## Fix Applied
Removed the assignment line `response["start_time"] = iso_utc(booking.created_at)` from the endpoint router.

## Why the Fix Works
Now the response keeps the correct `start_time` field generated by `serialize_booking()`.

## Regression Check
Re-ran the pytest integration test suite. All tests pass successfully with no warnings or regressions.

## Tests Added or Updated
N/A (Integration test asserts correct booking detail structures).
